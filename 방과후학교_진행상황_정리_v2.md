## 🔑 결정된 핵심 사항 (변경 금지)

### 시스템 설계 원칙

1. **정보보안 최우선** - 개인정보 유출 금지
2. **한 곳 수정 → 전체 자동 반영**
3. **매년 재사용** (연도만 바꿔서)
4. **단순한 수식** - 디버깅 어려운 수식 회피
5. **VBA 사용 안 함** - 순수 엑셀 수식만
6. **내부 참조만 사용** - 외부 참조는 깨지므로 절대 금지

### 수식 작성 규칙

* 일반 수식만 (배열 수식 금지)
* 사용 가능: SUMPRODUCT, COUNTIFS, INDEX/MATCH, \& 연산자
* 사용 금지: TEXTJOIN+IF, AGGREGATE (LibreOffice 호환 X)
* openpyxl이 함수명 소문자 변환 버그 (TEXTJOIN → textjoin) — \& 연산자로 회피

### 자격 우선순위 (어제 결정)

* 서식 3은 **1컬럼** (자격구분만)
* 한 학생이 2개 자격이면 → **2줄 입력**
* 셋팅 시트의 1순위(C23)·2순위(C24) 기준 자동 분배

### 고유번호 체계

* 형식: `연도 + 학년코드(A\~F) + 일련번호(001\~)`
* 예: 2026A001 = 2026년 1학년 1번째
* 학년코드: A=1학년, B=2학년, C=3학년, D=4학년, E=5학년, F=6학년

### 학생명단·자격 시트 구조 (40열)

* A\~F: 기본정보 (서식1 참조)
* G: 지원자격 통합 (예: "초3이용권+자유수강권")
* H: 자격상세
* I\~AF: 12개월 × (프로그램명, 사용금액) = 24컬럼

  * I=3월프로그램, J=3월금액, K=4월프로그램, L=4월금액, ...
* AG\~AN: 집계 (1순위·2순위 각각 자격명·한도·누적·잔여 4컬럼씩)
